# NixOS Contabo qcow2 Image Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a minimal, bootable NixOS 25.05 qcow2 image ready to upload to Contabo's Custom Images feature.

**Architecture:** A single `flake.nix` defines `nixosConfigurations.contabo`, composing three modules: NixOS's built-in `disk-image.nix` (image builder), `modules/contabo.nix` (Contabo-specific settings — VirtIO, BIOS boot, cloud-init), and `modules/user.nix` (empty stub for user customization). Building `.#nixosConfigurations.contabo.config.system.build.image` produces `result/nixos.qcow2`.

**Tech Stack:** Nix flakes, NixOS 25.05, `nixos/modules/virtualisation/disk-image.nix` (built-in image builder), GRUB (MBR), cloud-init, VirtIO drivers.

---

## Prerequisites

Nix must be installed with flakes enabled. Verify before starting:

```bash
nix --version
# Expected: nix (Nix) 2.x.x

nix show-config | grep experimental-features
# Expected: experimental-features = nix-command flakes
```

If flakes are not enabled, add this line to `~/.config/nix/nix.conf` (create if absent):

```
experimental-features = nix-command flakes
```

## Task 1: Create `modules/contabo.nix`

**Files:**
- Create: `modules/contabo.nix`

- [ ] **Step 1: Create the file**

```nix
# modules/contabo.nix
# Contabo-specific NixOS configuration.
# Do not edit this file — put customizations in user.nix instead.
{ pkgs, ... }: {

  # --- Image format ---
  image.format = "qcow2";
  image.efiSupport = false;   # Contabo VPS uses SeaBIOS (BIOS/MBR), not UEFI
  image.baseName = "nixos";   # produces result/nixos.qcow2 (not a versioned name)

  # --- Disk ---
  virtualisation.diskSize = 8192;  # 8 GiB in MiB; qcow2 is sparse

  # --- VirtIO drivers: required by Contabo ---
  boot.initrd.availableKernelModules = [
    "virtio_scsi"
    "virtio_net"
    "virtio_pci"
    "virtio_blk"
  ];

  # --- Networking ---
  networking.useDHCP = true;
  # Disable cloud-init network management to avoid conflict with dhcpcd
  services.cloud-init.network.enable = false;

  # --- SSH ---
  services.openssh.enable = true;
  services.openssh.settings.PermitRootLogin = "yes";

  # --- Cloud-init: Contabo injects SSH keys at first boot ---
  services.cloud-init.enable = true;

  # Required: set a hostname (cloud-init may override at first boot)
  networking.hostName = "contabo";

  system.stateVersion = "25.05";
}
```

- [ ] **Step 2: Commit**

```bash
git add modules/contabo.nix
git commit -m "feat: add contabo.nix — VirtIO, BIOS boot, cloud-init, SSH"
```

## Task 3: Create `flake.nix` and generate `flake.lock`

**Files:**
- Create: `flake.nix`
- Generated: `flake.lock`

- [ ] **Step 1: Create `flake.nix`**

```nix
# flake.nix
{
  description = "NixOS qcow2 image for Contabo VPS";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-25.05";
  };

  outputs = { self, nixpkgs }: {
    nixosConfigurations.contabo = nixpkgs.lib.nixosSystem {
      system = "x86_64-linux";
      modules = [
        "${nixpkgs}/nixos/modules/virtualisation/disk-image.nix"
        ./modules/contabo.nix
        ./modules/user.nix
      ];
    };
  };
}
```

- [ ] **Step 2: Lock the flake**

```bash
nix flake lock
```

Expected: `flake.lock` is created, resolving `nixpkgs` to a NixOS 25.05 revision.

- [ ] **Step 3: Commit**

```bash
git add flake.nix flake.lock
git commit -m "feat: add flake.nix with nixpkgs 25.05 and disk-image module"
```

```bash

```

Expected: prints a store path (e.g. `/nix/store/...-nixos.qcow2`), no errors.

If this fails with `attribute 'image' missing` — the `disk-image.nix` import path in `flake.nix` is wrong. Check the nixpkgs revision: `nix flake metadata` and verify the module exists at `${nixpkgs}/nixos/modules/virtualisation/disk-image.nix`.

If this fails with `attribute 'efiSupport' missing` — NixOS version does not have this option; pin nixpkgs more precisely to a 25.05 release tag.

## Upload Checklist (post-build)

1. Log in to the Contabo customer portal
2. Navigate to the VPS → Custom Images add-on (must be purchased)
3. Upload `result/nixos.qcow2`
4. OS type: **Linux** / version: **NixOS 25.05**
5. **Add your SSH public key** in the Contabo portal — cloud-init injects this at first boot. Do this before reinstalling or you will have no way in.
6. Enable **Cloud-Init toggle**
7. Reinstall VPS with the custom image
8. Wait for the reinstall to complete, then SSH in: `ssh root@<vps-ip>`
